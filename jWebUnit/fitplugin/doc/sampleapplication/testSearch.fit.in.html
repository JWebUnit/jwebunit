User Story: Searching the Fit Site

The fit site's welcome page should contain a single field form that allows the viewer to search the site.  The user enters text into the field, hits enter, and a search results page should then be presented containing a table of results.  Any pages containing the entered text will be displayed as navigable links within this table. A message is to be displayed after the results giving the number of matching pages and the total number of pages searched.  A link back to the welcom page should follow the search results.  A search field / form should be present abvove the search results, so the user can make another search directly from the results page.

Search this site:

Search results should look like this:

Testing the Story

We will organize the testing of this story into three acceptance tests to make them easier to understand and allow the functionality to be incrementally developed.  The tests we identify are initial form state on the welcome page, validation of static elements on the search results page (another search field, and the link back to the welcome page), and validation of the search results table.  You might organize them differently, or see tests that we have missed.

First we will test initial form state on the welcome page.  To do this, we establish the application url we are testing, and assert that there is a form present on the home page with a text input field named "search."

This last test is not how it would appear if we were testing against a static set of wiki pages under our control.  In that case, we would probably have search results and the search summary in separate html tables, and we would validate the exact contents of those tables rather than just testing for the one link being present.  As soon as Martin's name gets added somewhere, the above test will begin failing.
