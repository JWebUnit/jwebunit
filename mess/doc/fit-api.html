Navigation Commands

jWebUnit methods to navigate through a web application are explicitly supported on the WebFixture class:

base urlhttp://myserver:8080/myappSet the base url for the tested application.
begininfo.htmlBegin a series of interactions at a given page in the application.
presslinkText of LinkPress a link that contains the indicated text.
presslink idlinkIdPress a link with the indicated Id.
windowwindowIdOrTitleBegin interacting with indicated window.
base windowReturn to the base window.
pagepage.htmlBegin interaction with indicated page.
frameframeNameBegin interaction with indicated frame.

Form Entry and Submission Commands

Form interaction methods are also explicitly supported in the WebFixture class:

formformNameBegin working with named form - not needed if only one form on page.
enterformElementNameValue to enterProvide a value for a form element (use for radio groups and text boxes).
selectoptiondisplayedOptionValueChoose indicated select box option (by text displayed to user).
selectcheckboxcheckBoxNameSelect indicated checkbox (no support for specific value in WebFixture yet).
deselectcheckboxcheckBoxNameDeselect indicated checkbox.
presssubmitPress default submit button.
presssubmitbuttonNamePress submit button with indicated name.

Assertions / Checks

The WebFixture class allows access to all of the assertions on the jWebUnit WebTester class through use of reflection. The fit syntax is: | check | methodName | argument1 | argument2 | etc. The method name can be expressed in camel case, or you can put spaces between the words and the fixture will figure out the camel case.

Examples:
checktext presentexpected textDeterimine if text contained in exected response.
checkform element presentformElementNameDeterimine if expected form element is present.

Assertion methods on WebTester that end with "equals", like assertFormElementEquals, have a short-hand syntax that allows the "equals" to be left off. These methods are shown below.

checkform elementformElementNameexpected valueDeterimine if text contained in exected response.
checktitleexpected titleDeterimine if expected form element is present.
checkoptionselectBoxNameexpectedValueDeterimine if the indicated option of a drop down list is selected.

A few methods are shorthanded for convenience as well:

checklinklinkTextMaps to assertLinkPresentWithText(containedText).
checklink idlinkIdMaps to assertLinkPresent(id).

Debugging

A common way to diagnose unexpected results in jWebUnit is to dump the content of the current response in HttpUnitDialog. You can do this from fit as well - the content will go to system.err.

enterdump responseWrite content of current response to system.err.
