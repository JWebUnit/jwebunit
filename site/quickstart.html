Creating a TestCase

jWebUnit uses two approaches for creating test cases, inheritence and delegation. The simplest way is to inherit from WebTestCase rather than junit.framework.TestCase. 
import net.sourceforge.jwebunit.WebTestCase;

public class ExampleWebTestCase extends WebTestCase {
    public ExampleWebTestCase(String name) {
        super(name);
    }
}
The alternative is to include an instance of the WebTester class in you TestCase and delegate navigation and assertions to it. This is provided in case your need or prefer delegation. 
import junit.framework.TestCase;
import net.sourceforge.jwebunit.WebTester;

public class ExampleWebTestCase extends TestCase {
    private WebTester tester;

    public ExampleWebTestCase(String name) {
        super(name);
        tester = new WebTester();
    }
}
For the rest of the examples, we will use the inheritence approach.

Navigating Your Web Application

Navigation is a two step process. First, point your TestCase to your application, then access a particular resource. Let's say that your app is running on http://myserver:8080/myapp. To minimize the number of the times we must set this, we set this either in the constructor or in the setUp() method of our TestCase. 
    public ExampleWebTestCase(String name) {
        super(name);
        getTestContext().setBaseURL("http://myserver:8080/myapp");
    }
or 
    public void setUp() throws Exception {
        getTestContext().setBaseURL("http://myserver:8080/myapp");
    }
The TestContext encapsulates the configuration a TestCase. More information is availible in the Javadocs.

Now that your TestCase is pointed at your application, you can navigate to your particular resource. For example, you have a html page, info.html, located under your application root, then you can access it in your test method with the following code: 
    public void testInfoPage() {
        beginAt("/info.html");
    }

Submiting a Form

To submit a form, you will need to navigate to the form, fill out the input fields, then submit. 
    public void testFormSubmission() {
        beginAt("/info.html");
        setFormParameter("field1", "value1");
        setFormParameter("fieldn", "valuen");
        submit();
    }

Navigating Through a Series of Pages

To navigate through a series of resources, you can use beginAt, clickLink, and submit. 
    public void testNavigation() {
        beginAt("/somewhere");
        clickLink("link text");
        setFormParameter("field1", "value1");
        submit();
    }

Testing Your Pages

Once you have navigated to the page you wish to test, you can call the assertions provided by jWebUnit to verify it's correctness. 
    public void testCorrectness() {
        beginAt("/");
        assertTitleEquals("Home");
        clickLink("Sign Up");
        setFormParameter("uid", "jWebUnitUser");
        submit();
        assertTextInResponse("User successfully added."):
    }
For more information, consult the Javadocs.
